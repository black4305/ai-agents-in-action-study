## 1. 개요

LLM을 서비스에 통합할 때 선택할 수 있는 호스팅 방식은 크게 다섯 가지로 나뉜다:

| 방식 | 설명 | 대표 예시 |
|------|------|-----------|
| **로컬 추론** | 개인 PC/서버에서 직접 실행 | Ollama, LM Studio |
| **프로덕션 서빙** | GPU 서버에 추론 프레임워크 배포 | vLLM, SGLang, TensorRT-LLM |
| **관리형 OSS API** | 오픈소스 모델을 관리형으로 제공 | Groq, Together AI, Cerebras |
| **상용 API** | 프론티어 모델 API 호출 | OpenAI, Anthropic, Google |
| **GPU 렌탈** | 클라우드 GPU를 빌려 셀프호스팅 | RunPod, Vast.ai, Lambda |

2025~2026년 사이 API 가격은 약 80% 하락했으며, NVIDIA Blackwell(B200) 배포로 추론 비용이 최대 10배 감소했다.

## 3. 로컬 추론 엔진

### 3.1 llama.cpp

| 항목 | 내용 |
|------|------|
| **라이선스** | MIT (무료) |
| **성능** | CPU: 22~50 tok/s, GPU(RTX 4090): 80~200+ tok/s, Apple Silicon(M3 Max): 80~120 tok/s |
| **지원 모델** | 모든 GGUF 포맷 (Llama 4, Qwen3, Mistral, DeepSeek, Gemma 3, Phi-4) |
| **양자화** | 1.5-bit ~ 8-bit (Q2_K ~ Q8_0 등 다양한 변형) |

**VRAM 요구사항:**

| 모델 크기 | 최소 VRAM (Q4) | 권장 하드웨어 |
|-----------|---------------|-------------|
| 7~8B | 4~6 GB | RTX 3060 / M2 |
| 13B | 8~10 GB | RTX 4060 Ti 16GB |
| 32~34B | 16~20 GB | RTX 4090 / RTX 3090 |
| 70B | 35~40 GB | 듀얼 RTX 4090 |

**장점:** 최대 제어권, CPU+GPU 하이브리드 추론, Python 의존성 없음, 크로스 플랫폼, Apple Silicon Metal 지원 우수

**단점:** UI 없음(CLI), 동적 배칭 미지원, 수동 모델 다운로드/변환 필요

## 4. 프로덕션 서빙 프레임워크

### 4.1 vLLM

| 항목 | 내용 |
|------|------|
| **라이선스** | Apache 2.0 |
| **성능** | H100에서 1,000~2,500 tok/s, TGI 대비 최대 24배 처리량 |
| **지원 모델** | 50+ 아키텍처 (Llama 4, Qwen3, DeepSeek-V3/R1, Mistral, Gemma 등) |
| **핵심 기술** | PagedAttention (KV 캐시 낭비 60~80% → 4% 이하로 감소) |
| **기능** | 연속 배칭, FP8/INT8/AWQ/GPTQ 양자화, 추측 디코딩, 프리픽스 캐싱, Multi-LoRA, OpenAI 호환 API |

**장점:** 최대 커뮤니티, 가장 넓은 모델 지원, 활발한 개발, AWS/GCP/Azure ML 통합

**단점:** NVIDIA GPU 필수(AMD 개선 중), TensorRT-LLM 대비 피크 처리량 약간 낮음

**최적 용도:** 범용 프로덕션 서빙, 모델을 자주 교체하는 팀

---

### 4.2 SGLang

| 항목 | 내용 |
|------|------|
| **라이선스** | Apache 2.0 |
| **성능** | H100에서 ~16,200 tok/s (vLLM 대비 29% 빠름, 2026년 기준 최고 성능) |
| **핵심 기술** | RadixAttention (공유 프리픽스 워크로드에서 10~20% 추가 향상) |
| **기능** | 프리필-디코드 분리, 제로 오버헤드 CPU 스케줄러, 구조화된 출력 최적화, 추측 디코딩 |

**장점:** 2026년 기준 최고 처리량, RAG/챗봇/에이전트 워크로드에 최적, 구조화된 생성 우수

**단점:** vLLM 대비 작은 커뮤니티(빠르게 성장 중), 엣지 케이스 문서화 부족

**최적 용도:** RAG, 멀티턴 챗봇, 구조화된 데이터 추출, 에이전트 워크플로우

---

### 4.3 TensorRT-LLM

| 항목 | 내용 |
|------|------|
| **라이선스** | Apache 2.0 (NVIDIA 개발) |
| **성능** | H100에서 2,500~4,000+ tok/s (전 프레임워크 중 최고 피크), vLLM 대비 30~50% 높은 처리량 |
| **핵심 기술** | 컴파일 기반 커널 퓨전, FP4/FP8/INT8 양자화 |
| **제약** | NVIDIA GPU 전용, 모델별 컴파일 필요(수 시간 소요), 하드웨어별 빌드 |

**장점:** 최고 원시 처리량, 최적 GPU 활용, NVIDIA 공식 지원, FP4 양자화(Blackwell 전용)

**단점:** 모델 교체 시 재컴파일 필요, 설정 복잡, NVIDIA 종속

**최적 용도:** 고정 모델의 대규모 프로덕션, 최대 처리량이 필요한 NVIDIA 전용 환경

## 8. 비용 비교 분석

### 8.1 월간 토큰 볼륨별 비교

| 50M | ~$219 | ~$6 | $4,800~$5,400 | **관리형 OSS API** |

| 5B+ | ~$21,900 | ~$550 | $15,000~$25,000 | **셀프호스팅** |

### 8.2 손익분기점 분석

```
비용
 │
 │  ╱ 상용 API (GPT-4o)
 │ ╱
 │╱        ╱ 관리형 OSS API
 │        ╱
 │───────╱────── 셀프호스팅 (고정비 + 변동비)
 │      ╱
 ├─────┼─────┼──────┼──────┼────→ 토큰 볼륨
     100M  500M    1B    5B
                    ↑
              손익분기점 (약 1~5B 토큰/월)
```

**핵심 발견:**
- **월 10M 토큰 이하:** API가 거의 항상 저렴. 인프라 오버헤드 불필요
- **월 10M~500M 토큰:** 관리형 OSS API(Groq, DeepInfra)가 최적. 상용 API의 10~40배 저렴하면서 인프라 관리 불필요
- **월 1B 토큰 이상:** 셀프호스팅이 경제적. ROI 6~12개월
- **월 5B 토큰 이상:** 셀프호스팅 강력 권장. 60~80% 절감

### 8.3 GPU 활용률에 따른 비용 효율

| GPU 활용률 | H100 실효 비용 (1K 토큰당) | 비교 대상 |
|-----------|--------------------------|----------|
| 10% | $0.130 | 대부분 API보다 비경쟁 |
| 50% | $0.026 | GPT-4o급과 경쟁 가능 |
| 80%+ | $0.013 | 프리미엄 API 대비 우위, DeepInfra/Groq와는 비슷 |

> 최소 일 8,000건 이상의 대화가 있어야 셀프호스팅이 관리형 솔루션보다 경제적

### 8.4 3년 TCO (8x H100 시스템)

| 비용 항목 | 3년 합계 |
|----------|---------|
| 하드웨어 | ~$400,000 |

GPT-4o를 월 $65,000으로 사용 시 3년간 $2.34M → **셀프호스팅은 활용률 80%+ 시 18~24개월에 TCO 양전환**
